>>Om de file toch te kunnen laden heb ik deze functies tijdelijk 
>>uitgeschakeld (aangezien ik uit de website begreep dat u iets 
>>soortgelijks hebt gedaan met 3 andere unknown options). Het laden van de 
>>file werkt dan, maar als ik de library probeer te builden krijg ik de 
>>melding dat redboot_build niet bestaat.


I suppose you used ecos configtool for loading the .ecc ? You probably 
forgot to create the build-dirs, select build->generate build tree. Close 
configtool, the redboot_build directory should now be there. Cd into 
redboot_build and type make. Redboot should be build. If I am not 
mistaking, the .ecc in the example is made for grub booting. If you want to 
make a boot floppy, change the startup type to FLOPPY with configtool, save 
the ecc, generate the build tree again, and build redboot again. If all 
goes well, you should now have a redboot image that can be booted via 
floppy. Also notice that we use d a pc with two network interfaces, one is 
used by redboot for debugging, the other by ecos for ethercat. Make sure to 
configure things properly according to your setup.

If you really get stuck getting redboot operational on your system, the 
ecos mailinglists and archives are the place to look for help. This list is 
dedicated to the etherCAT master library.

Once you have redboot going, do the same thing in the library/ecos for 
building ecos together with the ethercat stack. Then you have to build the 
application in the topdir of the example (where you edited the makedefs 
file). If all goes well, an .elf file should now be present in the bin 
directory. You can now use redboot for loading and starting the application.


>>Ik veronderstel dat redboot.ecc is bedoeld een "OS" voor de target te 
>>kunnen bouwen en dat ecos.ecc in combinatie met ethercat.cxx  de 
>>applicatie moet worden. Klopt mijn denkwijze?


Nope, redboot.ecc is used for building the bootloader, ecos.ecc provides 
the OS with ethercat stack, build as a library that must be linked against 
the testapplication, as explained above.
